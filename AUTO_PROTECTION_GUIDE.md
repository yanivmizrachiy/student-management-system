# 🛡️ מנגנון הגנה אוטומטי - מדריך

## מטרה
למנוע הרס של קוד קיים בעת ביצוע שינויים. המנגנון רץ אוטומטית לפני ואחרי כל שינוי.

## איך זה עובד

### לפני כל שינוי:
המערכת בודקת:
- ✅ כל הקבצים הקריטיים תקינים
- ✅ כל האלמנטים הקריטיים קיימים
- ✅ כל הפונקציות הקריטיות קיימות
- ✅ כל החיבורים בין קבצים תקינים

### אחרי כל שינוי:
המערכת בודקת שוב:
- ✅ השינויים לא שברו דברים קיימים
- ✅ כל האלמנטים הקריטיים עדיין קיימים
- ✅ כל הפונקציות עדיין עובדות

## שימוש

### בדיקה ידנית:
```bash
python auto-protect.py
```

### לפני שינוי:
```bash
python safe-edit.py before
```

### אחרי שינוי:
```bash
python safe-edit.py after
```

## מה נבדק

### קבצים קריטיים:
1. **index.html** - דף הבית
2. **layer.html** - עמוד שכבה
3. **data.js** - מאגר נתונים
4. **auto-load-data.js** - טעינת נתונים
5. **shared-utils.js** - כלי עזר

### לכל קובץ נבדק:
- IDs קריטיים
- Scripts שנטענים
- Functions חשובות
- CSS Classes
- Strings קריטיים

## כללי זהב

1. ✅ **תמיד בדוק לפני שינוי** - `python safe-edit.py before`
2. ✅ **תמיד בדוק אחרי שינוי** - `python safe-edit.py after`
3. ✅ **אם יש שגיאה - אל תמשיך!** - תתקן קודם
4. ✅ **אם אתה לא בטוח - תשאל!**

## שימוש ב-AI (Cursor)

כשאתה מבקש מ-AI לתקן משהו, הוא:
1. יריץ `python safe-edit.py before` לפני השינוי
2. יבצע את התיקון
3. יריץ `python safe-edit.py after` אחרי השינוי
4. רק אם הכל תקין - יסיים
